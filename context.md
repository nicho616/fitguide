# Context - 需求诊断与对齐结论

针对本次健身动作浏览器与计划制定器（Fitness App）项目，经与用户诊断，达成以下核心结论：

## 1. 核心需求与场景 (User & Business Perspective)
* **用户目标**：最近准备开始健身，需要一个能快速检索动作、学习具体标准步骤并制定训练计划的工具。
* **部署方式（方案 B）**：应用需要部署到免费的云端托管平台（如 Vercel 或 GitHub Pages），以便用户能随时随地用手机访问，并支持安装到手机桌面（PWA 离线模式），方便在健身房使用。
* **难度与过滤**：默认展示所有难度等级 of 动作，支持通过分类筛选（身体部位、使用器械、目标肌群）和搜索。
* **计划制定**：支持将动作一键加入“今日计划列表”，并使用 `localStorage` 在本地持久化保存，避免刷新丢失。

## 2. 技术规格与视觉要求 (Technical & Visual Perspective)
* **技术选型**：纯原生静态 Web 开发（HTML + Vanilla CSS + JavaScript），无需复杂的编译环境，天然支持直接部署至 GitHub Pages / Vercel。
* **数据源**：使用 `hasaneyldrm/exercises-dataset` 提供的 1,324 项包含多语言指令的动作数据库。通过 Python 脚本将 `exercises.json` 下载至本地，页面内读取本地 JSON，保障高响应性。
* **语言偏好**：用户沟通与界面步骤完全显示为中文（`instructions.zh`）。
* **UI/UX 风格**：采用极具质感的暗黑运动风（拟物化与微卡片阴影），对手机手势和点击区域进行专项 Mobile-First 适配。
* **缺失资源兜底**：若原库 GIF/图片因版权缺失，使用精心设计的骨架屏和肌肉与器械图标进行美观降级。
