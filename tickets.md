# Tickets - 健身应用任务拆单与计划开发列表

本文件追踪 FitGuide 开发流程，每个 Ticket 将由 AI 在分支下独立完成并进行分步验证。

- [ ] **Ticket 1: 数据下载与准备**
  - 创建 Python 脚本 `download_data.py` 并从 GitHub 下载 `exercises.json` 到本地项目目录。
  - 验证：确保文件下载完成，且格式为正确的 JSON。

- [ ] **Ticket 2: 构建 HTML 基础框架 (index.html)**
  - 构建符合语义化 HTML5 的页面结构。
  - 包含导航头、搜索输入框、侧边栏筛选区、主要卡片网格和今日计划面板。
  - 验证：浏览器打开能看到基础 DOM 节点。

- [ ] **Ticket 3: 运动仪表盘 CSS 皮肤设计 (style.css)**
  - 编写自定义暗黑 UI 皮肤，使用柔和渐变与微卡片动效。
  - 针对手机触屏和不同尺寸屏幕做完全响应式布局处理。
  - 验证：基础页面视觉样式美观，布局符合预期。

- [ ] **Ticket 4: 数据加载与首屏惰性分页渲染 (app.js)**
  - 用原生 JS 的 fetch 获取本地 `exercises.json`。
  - 编写分页函数，只渲染前 20 个卡片，滚动到底部时自动加载更多。
  - 验证：数据成功加载且首屏极速呈现，滚动加载无卡顿。

- [ ] **Ticket 5: 交叉过滤与搜索算法实现 (app.js)**
  - 实现输入词在动作英文名及 target 中进行小写模糊匹配。
  - 结合侧边栏分类单选和器械多选进行多重交叉过滤。
  - 验证：搜索“push up”，同时筛选“chest”和“body weight”，能瞬间精确筛选。

- [ ] **Ticket 6: 制定训练计划与 localStorage 持久化 (app.js)**
  - 实现添加动作至计划、删除动作、清空计划逻辑。
  - 每次更改计划均写入 `localStorage`；打开网页时自动读取加载。
  - 提供“复制到剪贴板为 Obsidian 格式”按钮。
  - 验证：添加动作后刷新网页计划仍在，且能成功导出格式化 Markdown 文本。

- [ ] **Ticket 7: PWA 离线支持配置**
  - 创建 `manifest.json` 提供图标配置与全屏显示设置。
  - 编写 `sw.js` 实现对页面和 17MB 数据的强缓存，注册服务工作线程。
  - 验证：浏览器中显示服务工作线程注册成功，能模拟离线运行。

- [ ] **Ticket 8: 整体测试与视觉核验 (visual-verdict)**
  - 使用 Playwright 或内置分析机制在移动端分辨率和桌面端分辨率下核验页面排版、滚动流畅度和无报错日志。
  - 验证：输出测试结论和截图。

- [ ] **Ticket 9: 引导云端一键部署**
  - 提示用户运行 `npx -y vercel` 或推送到 GitHub 部署上线。
  - 验证：用户获得手机扫码可以直接打开的 HTTPS 网址。
